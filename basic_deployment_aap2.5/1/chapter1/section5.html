Containerized Installation LAB

  1. We are going to follow an All-in-one pattern approach for this setup.

  2. Once the lab is deployed and you are accessing the system via ssh terminal make sure to run everything as a normal user only and use the sudo command to perform any root-level operations.

  3. Install the required and useful utilities:

     # dnf install -y wget git-core rsync vim ansible-core

  4. Download the installer of the:

    # wget -O aap.tar.gz https://access.cdn.redhat.com/content/origin/files/sha256/48/48334cd93a9dffc0ecad2c390b8ff069d58a3c395e82ca38dce38650d44b4e61/ansible-automation-platform-containerized-setup-bundle-2.5-10-x86_64.tar.gz?

  5. Extract the aap.tar.gz file using the following command:

    tar -xvf aap.tar.gz

  6. Get the hostname from the system and copy it.

    hostname

  7. Enter the extracted folder and Open the inventory-growth file using the vim command and change the below

    aap.example.org → <system hostname>
<set your own> → redhat

<set your own> ← This value can be customized as per your need but remember to use the same while doing the login after the deployment.

  8. Once the inventory file is ready follow the below steps to do the installation:

    ansible-playbook -i inventory-growth ansible.containerized_installer.install

  9. Simply use the hostname from the lab deployment login page and try to access the URL with the browser on the port 443.
