Containerized Installation LAB

We will be using Product Enablement Containerized AAP 2.5 LAB for this deployment. Please be aware that setting up a RHEL 9 system in this lab will take about ten minutes. Following that, you can proceed with the installation procedures:

The below video demonstrates the step to step guide to do the installation of all components on a single system itself:

all the above steps followed in the video are shared below:

  1. We are going to follow an All-in-one pattern approach for the setup.

  2. Once the lab is deployed and you are accessing the system via ssh terminal make sure to run everything as a normal user only and use the sudo command to perform any root-level operations.

  3. Install the required and useful utilities:

     $ sudo dnf install -y wget git-core rsync vim ansible-core

  4. Go to the Installer link and Get the installer link for Ansible Automation Platform 2.5 Containerized Setup Bundle.

    Note: Please follow steps 4 and 5 one after another in order to avoid the link expiration error, as the copied URL has an expiration time. Reload the website, repeat step 4, and copy the link if you are having trouble.

  5. Download the installer file to the VM:

    $ wget -O aap.tar.gz "<paste-link-here taken from step 4>"

  6. Extract the aap.tar.gz file using the following command:

    $ tar -xvf aap.tar.gz

  7. Get the hostname from the system and copy it.

    $ hostname

  8. Using the cd command, enter the extracted folder. Then, use the vim command, open the inventory-growth file and make the following changes, wherever they appear throughout the entire inventory-growth file:

    $ cd <extracted-folder-from-step-6>
$ vim inventory-growth
aap.example.org → < paste system hostname>
<set your own> → redhat

<set your own> ← This value can be customized as per your need but remember to use the same while doing the login after the deployment.

    Note: Please click this link to learn how to use Vim basic operations.

  9. Once the inventory file is ready follow the below steps to do the installation:

    $ ansible-playbook -i inventory-growth ansible.containerized_installer.install

  10. Simply use the hostname from the lab deployment login page and try to access the URL from the browser on the port 443.

  11. Use the user admin and the password set during the time of installation. In the above video I set it to redhat.

    Note: Try to use stronger password while doing the enterprise deployment

  12. Provide the Red Hat login ID to get the subscription for attaching it to the AAP and 60-day subscription to Red Hat Ansible® Automation Platform or Red Hat Developer Subscription for Individuals subscription can be used for this lab.

  13. After Login run the Project sync for the Demo project to confirm the job is getting executed.
